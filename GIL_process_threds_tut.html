Understanding Python's GIL

Master Threads, Processes, Programs & The Global Interpreter Lock

üéØ Basics: Program, Process & Thread

Program: A collection of instructions written in a programming language. It's a static entity - just code on disk, not running.

Example:


                    # my_script.py (This is a program)
                    def greet(name):
                        print(f"Hello, {name}!")

                    greet("Alice")

The file my_script.py is a program. It doesn't do anything until you run it.

Key Point: A program is passive - it's just instructions. Nothing happens until it's executed.
Process: A running instance of a program. It has its own independent memory space, resources, and Python interpreter. Multiple processes from the same program run in isolation.

Example:


                    import os
                    import multiprocessing

                    def worker():
                        print(f"Process ID: {os.getpid()}")

                    if __name__ == '__main__':
                        p1 = multiprocessing.Process(target=worker)
                        p2 = multiprocessing.Process(target=worker)
                        p1.start()
                        p2.start()
                        
                    # Output: Two different Process IDs!
                    # Each has its own Python interpreter

Each process has its own:

  • Memory space (variables, objects)
  • Python interpreter
  • Resources (files, network connections)
  • Process ID (PID)
Key Point: Multiple processes = Multiple Python interpreters running independently. No GIL between processes!
Thread: A lightweight execution unit within a single process. All threads in a process share the same memory space and Python interpreter. Multiple threads execute concurrently but only one thread executes Python bytecode at a time due to GIL.

Example:


                    import threading

                    def worker():
                        print(f"Thread: {threading.current_thread().name}")

                    # Create 3 threads in the same process
                    t1 = threading.Thread(target=worker, name="Thread-1")
                    t2 = threading.Thread(target=worker, name="Thread-2")
                    t3 = threading.Thread(target=worker, name="Thread-3")

                    t1.start()
                    t2.start()
                    t3.start()

                    # Output: All threads in the same process!
                    # They share memory and one interpreter

All threads in a process share:

  • Same memory space (can access same variables)
  • Same Python interpreter
  • One GIL lock
Key Point: Multiple threads share resources but are serialized by GIL for CPU-bound work. Useful for IO-bound operations.

üîí Global Interpreter Lock (GIL)

GIL: A mutex (mutual exclusion lock) that protects access to Python objects in CPython. Only one thread at a time can execute Python bytecode, even on multi-core systems. This simplifies memory management but limits parallelism.

Why GIL Exists

Historical & Technical Reasons:

  • Memory Management: Python uses reference counting. GIL prevents multiple threads from corrupting the reference count simultaneously.
  • Implementation Simplicity: Single lock is simpler than fine-grained locking.
  • C Extension Compatibility: Most C extensions aren't thread-safe and rely on GIL.
  • Single-Threaded Performance: GIL makes single-threaded code faster (no locking overhead).

When to Use What:

‚úÖ Use Threading For:

  • Web scraping (IO-bound)
  • API calls (IO-bound)
  • File I/O (IO-bound)
  • Database queries (IO-bound)
  • Low resource overhead

‚úÖ Use Multiprocessing For:

  • Heavy calculations (CPU-bound)
  • Image processing (CPU-bound)
  • Data analysis (CPU-bound)
  • Machine learning (CPU-bound)
  • True parallelism needed

Key Insight:

  • CPU-Bound: Threading doesn't help (GIL serializes). Use multiprocessing.
  • IO-Bound: Threading helps (GIL released during wait). No need for multiprocessing overhead.
  • GIL: Not a bug, it's a design choice for memory safety and C compatibility.
  • Future: Python 3.13+ has experimental "free-threaded" mode that removes GIL.

üíª Code Examples

Example 1: CPU-Bound with Threading (Slow due to GIL)


                import threading
                import time

                def compute_prime(n):
                    # CPU-intensive task
                    count = 0
                    for i in range(2, n):
                        if is_prime(i):
                            count += 1
                    return count

                def is_prime(num):
                    if num < 2:
                        return False
                    for i in range(2, int(num**0.5)):
                        if num % i == 0:
                            return False
                    return True

                start = time.time()

                # Threading (GIL blocks - SLOW)
                threads = [
                    threading.Thread(target=compute_prime, args=(100000,)),
                    threading.Thread(target=compute_prime, args=(100000,))
                ]
                for t in threads: t.start()
                for t in threads: t.join()

                print(f"Threading: {time.time() - start:.2f}s")

Example 2: CPU-Bound with Multiprocessing (Fast - No GIL)


                import multiprocessing
                import time

                def compute_prime(n):
                    count = 0
                    for i in range(2, n):
                        if is_prime(i):
                            count += 1
                    return count

                start = time.time()

                # Multiprocessing (True parallelism - FAST)
                processes = [
                    multiprocessing.Process(target=compute_prime, args=(100000,)),
                    multiprocessing.Process(target=compute_prime, args=(100000,))
                ]
                for p in processes: p.start()
                for p in processes: p.join()

                print(f"Multiprocessing: {time.time() - start:.2f}s")

Example 3: IO-Bound with Threading (Fast - GIL Released)


                import threading
                import requests
                import time

                def fetch_url(url):
                    # IO-bound: GIL is released during network request
                    response = requests.get(url)
                    return response.status_code

                urls = ['http://api.example.com/1', 'http://api.example.com/2']

                start = time.time()

                # Threading (GIL released during IO - FAST)
                threads = [
                    threading.Thread(target=fetch_url, args=(url,))
                    for url in urls
                ]
                for t in threads: t.start()
                for t in threads: t.join()

                print(f"Threading IO-Bound: {time.time() - start:.2f}s")

üìù Key Takeaways

  • Program: Static code on disk
  • Process: Running instance with own memory & interpreter
  • Thread: Lightweight unit sharing memory & interpreter within a process
  • GIL: Mutex protecting Python object access - only ONE thread executes Python bytecode at a time
  • CPU-Bound: Use multiprocessing (bypasses GIL)
  • IO-Bound: Use threading (GIL released during IO)
  • GIL simplifies memory management but limits parallelism
  • Not a bug - a design choice for safety and C compatibility
